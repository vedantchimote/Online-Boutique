## CartService

Manages shopping cart operations for users.

<Tip>
  **What does this service do?** Think of it as the shopping cart manager. It lets you add items to a cart, view what's in the cart, and empty the cart when needed.
</Tip>

**Service Definition:**
```protobuf
service CartService {
    rpc AddItem(AddItemRequest) returns (Empty) {}
    rpc GetCart(GetCartRequest) returns (Cart) {}
    rpc EmptyCart(EmptyCartRequest) returns (Empty) {}
}
```

**Endpoint:** `cartservice:7070`

<Note>
  **What's an endpoint?** It's the address where this service lives. Think of it like a phone number - you need to know it to call the service.
</Note>

### AddItem

Adds an item to a user's shopping cart. If the item already exists, updates the quantity.

<Tip>
  **Use case:** When a user clicks "Add to Cart" on your website, you'd call this method.
</Tip>

**Request:** `AddItemRequest`
```protobuf
message AddItemRequest {
    string user_id = 1;    // Which user's cart
    CartItem item = 2;     // What to add
}
```

**Response:** `Empty` (no data returned, just success/failure)

**Example:**

<CodeGroup>
```bash cURL (Testing)
# This is great for testing without writing code!
grpcurl -plaintext -d '{
  "user_id": "user123",
  "item": {
    "product_id": "OLJCESPC7Z",
    "quantity": 2
  }
}' localhost:7070 hipstershop.CartService/AddItem
```

```go Go (Production)
// Use this in your Go application
req := &pb.AddItemRequest{
    UserId: "user123",
    Item: &pb.CartItem{
        ProductId: "OLJCESPC7Z",
        Quantity:  2,
    },
}
_, err := client.AddItem(ctx, req)
if err != nil {
    log.Printf("Failed to add item: %v", err)
}
```

```python Python (Production)
# Use this in your Python application
request = demo_pb2.AddItemRequest(
    user_id="user123",
    item=demo_pb2.CartItem(
        product_id="OLJCESPC7Z",
        quantity=2
    )
)
try:
    stub.AddItem(request)
    print("Item added successfully!")
except grpc.RpcError as e:
    print(f"Error: {e.details()}")
```
</CodeGroup>

**What happens:**
1. The service checks if the product is already in the user's cart
2. If yes, it updates the quantity
3. If no, it adds the product as a new item
4. The cart is saved to Redis (the database)

**Error Codes:**
- `INVALID_ARGUMENT`: Invalid product ID or quantity (e.g., negative quantity)
- `UNAVAILABLE`: Redis connection failed (database is down)

### GetCart

Retrieves a user's shopping cart.

**Request:** `GetCartRequest`
```protobuf
message GetCartRequest {
    string user_id = 1;
}
```

**Response:** `Cart`
```protobuf
message Cart {
    string user_id = 1;
    repeated CartItem items = 2;
}
```

**Example:**

<CodeGroup>
```bash cURL
grpcurl -plaintext -d '{
  "user_id": "user123"
}' localhost:7070 hipstershop.CartService/GetCart
```

```go Go

cart, err := client.GetCart(ctx, req)
if err != nil {
    log.Fatal(err)
}
fmt.Printf("Cart has %d items\n", len(cart.Items))
```

```python Python

print(f"Cart has {len(cart.items)} items")
```
</CodeGroup>

**Response Example:**

```json
      "quantity": 2
    },
    {
      "product_id": "66VCHSJNUP",
      "quantity": 1
    }
  ]
}
```

## Error Handling

All services use standard gRPC status codes. When something goes wrong, you'll get one of these codes to help you understand what happened.

<Tip>
  **Beginner Tip:** Always check for errors in your code! Don't assume API calls will always succeed. Networks fail, services go down, and users make mistakes.
</Tip>

| Code | When It Happens | What To Do | Example |
|------|-----------------|------------|---------|
| `OK` | Success! | Nothing - it worked | Request completed successfully |
| `INVALID_ARGUMENT` | Bad input data | Check your request data | Invalid product ID format |
| `NOT_FOUND` | Resource doesn't exist | Verify the ID exists | Product ID "XYZ" doesn't exist |
| `ALREADY_EXISTS` | Duplicate resource | Check if it already exists | Item already in cart |
| `PERMISSION_DENIED` | Not allowed | Check credentials/validation | Expired credit card |
| `UNAVAILABLE` | Service is down | Retry later or show error | Database connection failed |
| `INTERNAL` | Unexpected server error | Report to developers | Unhandled exception |

**Example Error Handling:**

<Tabs>
  <Tab title="Go">
    ```go
    resp, err := client.GetProduct(ctx, req)
    if err != nil {
        // Check what kind of error it is
        if status.Code(err) == codes.NotFound {
            fmt.Println("Product not found - maybe it was deleted?")
        } else if status.Code(err) == codes.InvalidArgument {
            fmt.Println("Invalid product ID - check your input")
        } else {
            fmt.Printf("Unexpected error: %v\n", err)
        }
        return err
    }
    // Success! Use the response
    fmt.Printf("Product: %s\n", resp.Name)
    ```
  </Tab>

  <Tab title="Python">
    ```python
    try:
        response = stub.GetProduct(request)
        print(f"Product: {response.name}")
    except grpc.RpcError as e:
        # Check the error code
        if e.code() == grpc.StatusCode.NOT_FOUND:
            print("Product not found - maybe it was deleted?")
        elif e.code() == grpc.StatusCode.INVALID_ARGUMENT:
            print("Invalid product ID - check your input")
        else:
            print(f"Unexpected error: {e.details()}")
    ```
  </Tab>

  <Tab title="Node.js">
    ```javascript
    client.getProduct({ id: productId }, (err, response) => {
        if (err) {
            // Check the error code
            if (err.code === grpc.status.NOT_FOUND) {
                console.log("Product not found - maybe it was deleted?");
            } else if (err.code === grpc.status.INVALID_ARGUMENT) {
                console.log("Invalid product ID - check your input");
            } else {
                console.log(`Unexpected error: ${err.message}`);
            }
            return;
        }
        // Success! Use the response
        console.log(`Product: ${response.name}`);
    });
    ```
  </Tab>
</Tabs>

<Warning>
  **Common Mistake:** Don't ignore errors! Always check if an API call succeeded before using the response data.
</Warning>

## Common Issues & Troubleshooting

<AccordionGroup>
  <Accordion title="Connection Refused Error">
    **Problem:** Can't connect to the service
    
    **Solutions:**
    - Check if the service is running: `kubectl get pods`
    - Verify the endpoint address and port
    - Make sure you're using the correct namespace
    - Check if there's a firewall blocking the connection
  </Accordion>

  <Accordion title="Invalid Argument Error">
    **Problem:** Getting `INVALID_ARGUMENT` errors
    
    **Solutions:**
    - Double-check your request data matches the proto definition
    - Verify all required fields are included
    - Check data types (string vs int, etc.)
    - Look at the error message for specific details
  </Accordion>

  <Accordion title="Deadline Exceeded Error">
    **Problem:** Request times out
    
    **Solutions:**
    - Increase the timeout in your client code
    - Check if the service is overloaded
    - Verify network connectivity
    - Look for slow database queries
  </Accordion>

  <Accordion title="Not Found Error">
    **Problem:** Resource doesn't exist
    
    **Solutions:**
    - Verify the ID is correct (check for typos)
    - Make sure the resource was created successfully
    - Check if it was deleted
    - Try listing all resources to see what exists
  </Accordion>

  <Accordion title="Can't Generate Client Code">
    **Problem:** `protoc` command fails
    
    **Solutions:**
    - Install Protocol Buffers compiler: `brew install protobuf` (Mac) or download from [protobuf releases](https://github.com/protocolbuffers/protobuf/releases)
    - Install language-specific plugins (e.g., `protoc-gen-go`)
    - Check your proto file syntax
    - Verify import paths are correct
  </Accordion>
</AccordionGroup>
